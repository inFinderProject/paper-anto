# Workflow 07: 레이아웃 매핑 워크플로 가이드

> 역할: 04 확정 목차와 05 슬라이드 텍스트를 기반으로, **각 슬라이드를 레이아웃 패턴(구조)와 간단 와이어프레임에 매핑**하는 단계입니다.  
> 이 단계의 핵심 산출물은 MD 기반의 “슬라이드 → 패턴 코드 → 와이어프레임 메모”입니다.

## 1. 입력과 출력

- 입력:
  - 04 2차 목차 확정본  
    - `projects/<project-name>/04-outline-second/outline-v2.md`
  - 05 슬라이드 설계·콘텐츠 초안  
    - `projects/<project-name>/05-*/section*-*.md`
  - 레이아웃 패턴 라이브러리  
    - `layout-patterns/` (예: `layout.jsx`, 패턴 설명 문서)

- 출력:
  - 슬라이드 → 레이아웃 매핑 문서(MD, 필수)  
    - 섹션별: `projects/<project-name>/07-layout/sectionX-layout-mapping.md`  
      - 각 섹션의 슬라이드ID 별로 PatternCategory/PatternCode/간단 와이어프레임 메모 정리  
    - 전체 요약: `projects/<project-name>/07-layout/00-slide-layout-map.md`  
      - `섹션/슬라이드ID → 레이아웃 카테고리·패턴코드·비고` 표
  - (선택) 레이아웃 시안/React 아티팩트  
    - 예: `07-layout/section1-layout-visual.jsx`, `07-layout/S2-1-layout.jsx` 등 16:9 구조 미리보기  
    - 존재하면 좋지만, **MD 매핑 문서가 진짜 기준(source of truth)** 입니다.

## 2. 워크플로 위치

- 00 인테이크 → 01 1차 목차 → 02 리서치 → 03 요약·브리프 압축 → 04 2차 목차 → 05 슬라이드 작성 → 06 내용 품질·정합성 점검 → **07 레이아웃 매핑** → 08 디자인 가이드·템플릿 → 09 실제 PPT 제작

## 3. 운영 원칙 (초안)

- 07 단계에서는 **슬라이드 하나당 레이아웃 패턴 하나**를 우선 원칙으로 하고,  
  필요할 때만 2개 패턴을 조합해서 사용합니다.
- 매핑 기준(정보 구조 관점):
  - 비교·표 구조 → `비교형/나열형` 패턴
  - 프로세스·타임라인 → `시간형` 패턴
  - 전략 맵·포지셔닝 → `구조형` 패턴
  - 핵심 메시지 강조 → `강조형` 패턴
  - 데이터·표·그래프 → `데이터형` 패턴
- 매핑 결과는 **반드시 MD 문서**(`sectionX-layout-mapping.md`, `00-slide-layout-map.md`)에서 관리하며,  
  이후 08 단계 디자인 가이드·템플릿, 09 단계 실제 PPT 제작은 이 문서를 기준으로 진행합니다.
