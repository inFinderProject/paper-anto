# Workflow 08: PPT 디자인 가이드·템플릿 워크플로 가이드

> 역할: 브랜드(퀴즈노스)와 병원 환경에 맞는 **PPT 디자인 가이드와 템플릿 구조를 먼저 정의**해, 09단계 실제 슬라이드 제작 전에 시각·구조적 기준을 통일하는 단계입니다.

## 1. 입력과 출력

- 입력:
  - 04 2차 목차 확정본  
    - `projects/<project-name>/04-outline-second/outline-v2.md`
  - 05 섹션별 슬라이드 텍스트 초안  
    - `projects/<project-name>/05-*/section*-*.md`
  - 07 레이아웃 매핑 결과  
    - `projects/<project-name>/07-layout/00-slide-layout-map.md`  
    - `projects/<project-name>/07-layout/section*-layout-mapping.md`
  - 브랜드·환경 레퍼런스  
    - 퀴즈노스 브랜드 자료(R2), 병원 UI/브랜딩 규정(있다면)

- 출력:
  - PPT 디자인 가이드 문서  
    - 예: `projects/<project-name>/08-design-guide/00-ppt-design-guide.md`
    - 내용:
      - 컬러 팔레트 (브랜드 기본색·보조색·상태색)
      - 타이포그래피(제목/본문/캡션 폰트·크기·줄간격)
      - 그리드·마진·레이아웃 규칙(제목 위치, 본문 영역, 여백)
      - 아이콘·일러스트 스타일, 사진 스타일
      - 표·그래프 스타일 (선·채우기·레이블 규칙)
      - 챕터 구분 슬라이드·표지·엔딩 슬라이드 구조
  - PPT 템플릿 구조 정의  
    - 예: `08-design-guide/template-structure.md`
    - 내용: "기본 마스터 슬라이드 타입" 목록
      - 표지, 섹션 구분, 제목+본문, 2열 비교, 3열 카드, 표/그래프, 타임라인, 로드맵 등

## 2. 워크플로 위치

- 00 인테이크 → 01 1차 목차 → 02 리서치 → 03 요약·브리프 압축 → 04 2차 목차 → 05 슬라이드 작성 → 06 내용 품질·정합성 점검 → 07 레이아웃 매핑 → **08 PPT 디자인 가이드·템플릿** → 09 실제 PPT 제작·산출물

## 3. 운영 원칙 (초안)

- 08 단계에서는 **브랜드·병원 환경에 맞는 시각 언어를 먼저 고정**합니다.
  - 퀴즈노스 브랜드 컬러(Toasty·Premium·Healthy 계열), 병원 UI 색(있다면)의 조화
  - 본문 가독성을 최우선(16:9, 회의실 프로젝터 기준)
- 디자인 가이드는:
  - 실제 PPT 제작 전 디자이너·작성자가 공유하는 **단일 기준 문서**로 관리하고,
  - 프로젝트별로 `08-design-guide/` 아래에 버전관리(`v1`, `v2` 등)를 남깁니다.
- 템플릿 구조는:
  - 07 단계의 `slide-layout-map`과 섹션별 레이아웃 매핑을 참고해
  - "어떤 슬라이드 유형(패턴 코드)이 마스터 템플릿으로 필요한지"를 역산하여 정의합니다.
  - 이때 **새 패턴을 발명하지 않고**, 07에서 이미 결정한 패턴 집합 안에서 색·폰트·스타일만 결정하는 것을 원칙으로 합니다.
