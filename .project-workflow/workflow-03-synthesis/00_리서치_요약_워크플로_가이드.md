# Workflow 03: 리서치 요약·브리프 압축 워크플로 가이드

> 역할: 02 리서치 결과를 **한눈에 볼 수 있는 요약 브리프 + 리서치 참조 맵**으로 정리해, 2차 목차(Workflow 04)를 설계하기 직전의 정리 단계입니다.

## 1. 입력과 출력

- 입력:
  - 00 인테이크 브리프 인스턴스 (`00-intake/brief.md`)
  - 01 1차 목차(가설 구조)
  - `02-research/` 아래 R1~R6 리서치 노트(md·pdf)
- 출력:
  - `03-synthesis/00-research-synthesis.md`:
    - 요약 브리프(5~7줄)
    - R1~R6 리서치 참조 맵
    - R코드별 핵심 인사이트 3~5개

## 2. 워크플로 위치

- 00 인테이크 → 01 1차 목차 구성 → 02 리서치 → **03 리서치 요약·브리프 압축** → 04 2차 목차 구성

## 3. 운영 원칙 (초안)

- 브리프 전체를 그대로 들고 가기보다, **요약 브리프만 보고도 2차 목차를 설계할 수 있을 정도로 압축**합니다.
- R1~R6 리서치 노트는:
  - 파일·섹션별로 어디에 쓸지(어느 섹션에 연결되는지)를 참조 맵에 먼저 표시하고,
  - 각 R코드당 3~5개의 “꼭 써야 할 인사이트”만 골라 적어둡니다.
- 이후 04 단계(2차 목차)는 이 `00-research-synthesis.md`를 메인 입력으로 사용하며,  
  실제 목차 설계/슬라이드 구조 논의 시에는 **가능하면 이 문서만 보면서 의사결정**하는 것을 기본으로 합니다.
