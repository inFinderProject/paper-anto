# Workflow 09: 실제 PPT 제작·산출물 패키징 워크플로 가이드

> 역할: 04~08 단계에서 준비한 목차·콘텐츠·레이아웃·디자인 가이드를 바탕으로 **실제 PPT 파일과 부가 산출물(PDF/요약본)을 제작·패키징**하는 단계입니다.

## 1. 입력과 출력

- 입력:
  - 04 2차 목차 확정본  
    - `projects/<project-name>/04-outline-second/outline-v2.md`
  - 05 섹션별 슬라이드 텍스트 초안  
    - `projects/<project-name>/05-*/section*-*.md`
  - 07 슬라이드 → 레이아웃 매핑  
    - `projects/<project-name>/07-layout/00-slide-layout-map.md`
  - 08 PPT 디자인 가이드·템플릿 구조  
    - `projects/<project-name>/08-design-guide/*.md`

- 출력:
  - 최종 PPT 파일  
    - 예: `projects/<project-name>/09-deliverables/퀴즈노스_인하대병원점_제안서_v1.pptx`
  - PDF 버전  
    - 예: `.../퀴즈노스_인하대병원점_제안서_v1.pdf`
  - (선택) 1~2장 요약본, 부록 정리본 등  
    - 예: `.../요약본_2p.pdf`, `.../부록_브리프_리서치요약.md`

## 2. 워크플로 위치

- 00 인테이크 → 01 1차 목차 → 02 리서치 → 03 요약·브리프 압축 → 04 2차 목차 → 05 슬라이드 작성 → 06 내용 품질·정합성 점검 → 07 레이아웃·디자인 → 08 PPT 디자인 가이드·템플릿 → **09 실제 PPT 제작·산출물 패키징**

## 3. 운영 원칙 (초안)

- 09 단계에서는 **새로운 내용·구조를 추가하지 않고**,  
  04~08에서 합의된 내용·레이아웃(07)·디자인 가이드/템플릿(08)을 충실히 구현하는 데 집중합니다.
- 실제 작업 단계를 권장 순서로 나누면:
  1. 템플릿 선택/셋업: 08단계에서 정의한 마스터 슬라이드·폰트·컬러 적용
  2. 슬라이드 인스턴스 생성: 07의 `slide-layout-map`/섹션별 매핑과 05 텍스트를 보고 섹션1~7 슬라이드 채우기
  3. 시각 QA: 여백·폰트 크기·가독성·정렬 점검, 필요 시 경미한 텍스트/레이아웃 조정
  4. 산출물 패키징: PPTX 저장, PDF export, 요약본/부록 생성, 파일명·버전/날짜 규칙 적용
- 최종 파일들은 프로젝트별 `09-deliverables/` 폴더에만 모아두고,  
  실제 클라이언트 전달 시에는 이 폴더의 내용만 공유하도록 합니다.
