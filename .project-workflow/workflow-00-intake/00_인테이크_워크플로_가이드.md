# Workflow 00: 인테이크 워크플로 가이드

> 역할: 프로젝트 시작 시 **요청 정보를 구조적으로 수집**하고, 이후 단계(1차 목차 → 리서치 → 2차 목차)의 기반이 되는 브리프 데이터를 만드는 단계입니다.

## 1. 연결되는 자산

- `ppt-intake-app/`  
  - Next.js + shadcn/ui로 구현된 4단계 인테이크 폼  
  - 출력: 프로젝트 메타데이터, JSON/YAML, 브리프 초안
- `brief-templates/`  
  - 프로젝트 유형별 브리프 템플릿(수주/내부보고/IR/마케팅)

## 2. 워크플로 위치

- 00 인테이크 → **01 1차 목차 구성** → 02 리서치 → 03 리서치 요약·브리프 압축 → 04 2차 목차 구성

## 3. 운영 원칙 (초안)

- 모든 프로젝트는 반드시 `ppt-intake-app`을 통해 인테이크를 생성한다.
- 브리프는 `brief-templates/` 구조에 맞춰 저장·관리하고, 프로젝트별 인스턴스는 `projects/<project-name>/00-intake/brief.md` 형태로 남긴다.
- 인테이크 결과에서 바로 1차 목차 구성을 위한 요약을 추출하고, 이후 03 단계에서 사용할 **요약 브리프용 핵심 정보**(목표·성공 기준·상권 한줄 정의·제약)를 미리 염두에 두고 정리한다.
