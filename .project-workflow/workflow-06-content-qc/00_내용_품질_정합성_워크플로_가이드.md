# Workflow 06: 내용 품질·정합성 점검 워크플로 가이드

> 역할: 04 확정 목차와 05 슬라이드 초안을 기준으로, **섹션 간 내용 품질·용어·숫자·스토리 흐름의 정합성을 점검·정리**하는 단계입니다.

## 1. 입력과 출력

- 입력:
  - 04 2차 목차 확정본  
    - `projects/<project-name>/04-outline-second/outline-v2.md`
  - 05 슬라이드 설계·콘텐츠 초안  
    - `projects/<project-name>/05-*/section*-*.md` (섹션1~7)
  - 03 리서치 요약  
    - `projects/<project-name>/03-synthesis/00-research-synthesis.md`

- 출력:
  - QC 체크 결과 정리 문서(예: `projects/<project-name>/06-qc/00-content-review.md`)  
    - 용어·명칭 통일 규칙
    - 숫자·KPI·목표 정합성 메모
    - 섹션별 스토리 흐름·중복/누락 정리
    - 수정 필요 TODO 리스트

## 2. 워크플로 위치

- 00 인테이크 → 01 1차 목차 → 02 리서치 → 03 요약·브리프 압축 → 04 2차 목차 → 05 슬라이드 작성 → **06 내용 품질·정합성 점검** → 07 레이아웃·디자인

## 3. 운영 원칙 (초안)

- 06 단계에서는 **새 내용을 추가하기보다는**, 이미 작성된 슬라이드를 기준으로:
  - 용어·명칭 통일 (타깃 명칭, 숫자 표기, 키워드 스펠링 등)
  - 숫자·목표 일관성 (인지도 20%, 매출 200%, KPI·시나리오 수치)
  - R코드 인사이트 인용의 충돌 여부
  - 섹션 간 스토리 흐름·중복·누락
  를 점검·수정합니다.
- 점검 결과는 프로젝트별로 `06-qc/00-content-review.md`에 누적 관리하고,  
  슬라이드 파일 자체 수정은 **명확한 이유와 함께** 단계별로 진행합니다.
