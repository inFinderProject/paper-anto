```markdown
[파일명](../../research/파일명.md)  ← 상대 경로로 링크
```

```
project/02-research/README.md (현재)
↓
../../research/파일명.md (research 폴더)
```

### 6.2 리서치 파일 분류 규칙 (향후)

프로젝트 진행 중 새로운 리서치 자료가 생기면, 다음 규칙을 따라 관리:

```
project/02-research/
├── README.md (네비게이션)
├── RESEARCH-STATUS.md (완성도 체크리스트)
├── Chapter-[숫자]/
│   ├── [Chapter 제목] Summary.md
│   ├── resources.md (해당 Chapter의 리서치 파일 목록)
│   └── assets/ (이미지, 요약 문서 등)
└── assets/
    ├── images/
    ├── templates/
    └── references/
```

### 6.3 MCP 기반 도구 활용 (CLAUDE.md 준수)

> **참고**: CLAUDE.md에 따라, 사실 기반 여부가 애매하거나 기준이 불명확할 경우
> - 일반 웹 검색 대신 **MCP 기반 도구**를 우선 활용
> - 예: `mcp__duckduckgo__search`, `mcp__duckduckgo__fetch_content` 등

리서치 자료 보충이 필요할 때:
1. 기존 리서치 파일들(`/research/`) 우선 활용
2. 부족한 경우 → MCP 기반 도구로 추가 정보 수집
