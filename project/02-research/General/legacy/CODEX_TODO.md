# CODEX To‑Do List (ANTO 보고서 제작)

본 문서는 ANTO 보고서 콘텐츠 제작을 위한 실행 지침과 섹션별 작업 목록(To‑Do)을 정의합니다. 전체 흐름은 작업 워크플로우(`work-contents/workflow.md`)와 목차(`work-contents/outline.md` 및 `outline.md`)를 기준으로 진행합니다.

## 목적과 범위
- 목적: `목차`에 맞춘 섹션별 콘텐츠를 체계적으로 생산·검수·승인.
- 범위: 초안 작성 → 근거 데이터 정리 → 팩트체크 → 편집 통합 → 최종 승인.
- 산출물: 섹션별 마크다운 원고, 표/차트용 데이터, 최종 통합본.

## 기준 문서와 참고 자료
- 워크플로우: `work-contents/workflow.md:1`
- 목차: `work-contents/outline.md:1`, `outline.md:1`
- 프롬프트 템플릿: `work-contents/PROMPT_TEMPLATES.md:1`
- 기존 분석 콘텐츠: `안토_종합분석보고서.md:1`, `work-contents/안토_종합분석보고서.md:1`
- 인사/운영 분석: `안토_인사팀_업무보고서_분석.md:1`, `work-contents/안토_인사팀_업무보고서_분석.md:1`
- 리서치 원본: `research/` 내 DOCX/PDF 일체
- 브랜드북/시안: `appendix/` 내 PDF/JPG
- 뉴스/스크래핑: `news_Anto*.{csv,tsv,json}`, `news_Anto.md`

## 파일 구조와 명명 규칙
- 초안 산출물 경로: `work-contents/content/`
  - 파일명: `SNN_섹션명.md` (예: `S02_운영_현황.md`)
  - 표/차트 데이터: `work-contents/data/SNN_섹션명.csv` (필요 시)
- 최종 통합본: `work-contents/ANTO_Report_Final.md`
- 기존 `articles/NN-News.md`는 뉴스 기사 보관용으로 유지(혼용 금지).

## 작업 순서(Phase)
1) 준비: 자료 매핑·우선순위 설정(워크플로우 Phase 1 준수)
2) 섹션 작성: 핵심 메시지 → 근거 수집 → 초안 → 팩트체크(Phase 2)
3) 편집/통합: 톤·스타일·표기 통일, 표/차트 삽입
4) 리뷰/승인: 내부 리뷰 → 오너 승인 → 잠금
5) 산출/백업: 최종본, 데이터, 변경이력 정리

## 공통 작성 가이드
- 톤앤매너: 전문적·간결·검증 가능한 비즈니스 리포트 문체
- 형식: 2~4 수준 헤더 사용, 표/불릿 적극 활용, 그림 캡션 필수
- 출처: 수치·주장 옆에 출처 파일/페이지 병기(예: (research/01_...pdf, p.12))
- 데이터: CSV 원본과 표 변환본 동시 관리, 단위·기간 명시
- 금칙: 미확인 추정치 단정 금지, 브랜드 키워드 오용 금지
