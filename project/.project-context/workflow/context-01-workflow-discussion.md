나는 시각 디자인과 기획 업무 영역에 많은 경험이 있어. 제안서, 보고서, 마케팅 기획 등에 대한 문서를 돈을 받고 프로젝트 성으로 업무를 추진하고 있는데, 이것을 효과적으로 업무를 추진하기 위해서 AI를 활용하려고 해. 

현재도 많은 도움을 받고 있지만, 현실적인 정보나 맥락 등에 대한 내용을 모두 AI에게 전달할 수가 없어서 내가 작업을 60%는 하고 있어. 리서치 부분은 AI의 도움을 정말 많이 받고 있는데, 워크플로를 AI 기반으로 더 촘촘하게 구체화하여 많은 프로젝트를 받더라도 업무를 추진할 수 있도록 하기 위하여 워크플로를 기획을 요청하고자 해.

그리고 AI를 활용할 때 PPTX 만들어주는 HTMLtoPPT 가 있지만(스킬) 아직 돈을 받고 상품화 하기에는 어려움이 있어. 하지만 초안의 형태 구성 - 레이아웃 구조에 따른 콘텐츠 배치, 콘텐츠의 내용에 따라 페이지 구성 등은 미리 확인해볼 수 있을 것 같아.

그리고 만드는 것은 내가 계속 업데이트하면서 나아가면 될 것 같고. 여기에서 더 상세한 워크플로와 촘촘한 워크플로를 기획해서 나에게 안내해줘.

좋아요. 시각 디자인과 기획 전문가로서 AI를 활용한 워크플로 최적화를 원하시는 거군요.

효과적인 워크플로를 설계하기 전에 몇 가지 확인하고 싶습니다.

**현재 업무 프로세스 파악**

1. **주로 다루시는 문서 유형**은 어떤 비중인가요?
   - 제안서 / 보고서 / 마케팅 기획서 / 기타

2. **프로젝트당 평균 소요 시간**은 어느 정도인가요?
   - 예: 제안서 1건에 3일, 마케팅 기획서 1건에 1주일 등

   - 예: 최종 디자인 작업, 클라이언트 맥락 반영, 스토리라인 조정 등

4. **AI에게 전달하기 어려운 "현실적인 정보나 맥락"**이란 구체적으로 어떤 것들인가요?
   - 예: 클라이언트 내부 정치, 이전 프로젝트 히스토리, 업계 특수 용어 등

5. **한 달에 몇 개의 프로젝트**를 진행하시고, 목표는 몇 개까지 늘리고 싶으신가요?

이 정보를 바탕으로 실제로 적용 가능한, 구체적인 워크플로를 설계해 드릴게요.

## AI 기반 제안서/기획서 워크플로 시스템

### 전체 구조

```
[인풋 시스템] → [리서치] → [구조화] → [콘텐츠 생성] → [레이아웃] → [디자인] → [납품]
     ↓              ↓           ↓            ↓             ↓           ↓
  템플릿화        AI 주도     AI 주도      AI+검토       AI 초안      사람 집중
```

---

### Phase 1. 인풋 시스템 (맥락 전달 문제 해결)

**문제**: 매번 AI에게 맥락을 설명하는 데 시간 소모

**해결**: 프로젝트 브리프 템플릿 + 클라이언트 프로파일 DB

```
📁 프로젝트 브리프 템플릿
├── 클라이언트 기본정보 (업종, 규모, 의사결정자 성향)
├── 프로젝트 목적 (제안서/기획서, 최종 목표)
├── 핵심 메시지 (반드시 들어가야 할 내용 3가지)
├── 금기사항 (피해야 할 표현, 경쟁사 언급 등)
├── 레퍼런스 (선호하는 스타일, 이전 작업물)
└── 일정 & 예산
```

**실행 방법**:
1. 클라이언트별 프로파일을 Notion/문서로 관리
2. 새 프로젝트 시작 시 브리프 템플릿 작성 (15분)
3. 이 브리프를 AI 대화 시작점으로 활용

---

### Phase 2. 리서치 (현재 잘 활용 중 → 더 체계화)

```
리서치 체크리스트
├── 시장/산업 동향
├── 경쟁사 분석
├── 타겟 고객 인사이트
├── 관련 데이터/통계
└── 벤치마킹 사례
```

**AI 활용 포인트**:
- Claude에게 리서치 요청 시 "브리프 기반"으로 요청
- 결과물을 구조화된 형태로 받기 (표, 요약, 핵심 인사이트)

---

### Phase 3. 구조화 (스토리라인 + 페이지 구성)

**AI에게 요청할 것**:
```
"이 브리프와 리서치 결과를 바탕으로
1. 전체 스토리라인 (논리 흐름)
2. 페이지별 구성안 (제목 + 핵심 메시지 + 필요 콘텐츠 유형)
3. 각 페이지의 목적과 다음 페이지로의 연결고리
를 제안해줘"
```

**산출물 예시**:
| 페이지 | 제목 | 핵심 메시지 | 콘텐츠 유형 | 목적 |
|--------|------|-------------|-------------|------|
| 1 | 표지 | - | 이미지+타이틀 | 첫인상 |
| 2 | 문제 정의 | "현재 이런 어려움이..." | 데이터+아이콘 | 공감 유도 |
| 3 | 해결책 | "우리의 접근법은..." | 다이어그램 | 신뢰 구축 |

---

### Phase 4. 콘텐츠 생성

**AI에게 페이지별로 요청**:
- 헤드라인 카피 3가지 옵션
- 본문 텍스트 (분량 지정)
- 데이터 시각화 제안
- 이미지 키워드/방향성

**당신의 역할**: 선택 + 맥락에 맞게 조정 (결정만 하면 됨)

---

### Phase 5. 레이아웃 (여기가 핵심 - 시간 절약 포인트)

**AI 활용 방법**:

```
요청 예시:
"이 페이지의 콘텐츠:
- 헤드라인: ○○○
- 서브텍스트: ○○○
- 데이터: 3개 항목 비교
- 강조점: ○○○

이 내용에 맞는 레이아웃 옵션 3가지를 
와이어프레임 형태로 보여줘 (HTML/React로)"
```

**산출물**: 
- React 아티팩트로 레이아웃 시각화
- 비율, 배치, 시각적 위계 확인
- 마음에 드는 구조 선택 → 디자인 작업 시 참고

---

### Phase 6. 디자인 작업 (당신의 전문 영역)

**AI가 준비해둔 것**:
- 확정된 레이아웃 구조
- 모든 텍스트 콘텐츠
- 이미지 방향성/키워드
- 컬러 팔레트 제안

**당신이 할 것**:
- 최종 비주얼 완성 (전문성 발휘)
- 클라이언트 톤앤매너 반영
- 퀄리티 컨트롤
