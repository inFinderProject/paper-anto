# 🔄 프로젝트 컨텍스트: AI 기반 제안서/기획서 워크플로 시스템 구축

---

## 프로젝트 개요

**목표:** 시각 디자인 + 기획 전문가가 AI를 활용하여 제안서/기획서 프로젝트를 월 1개 → 월 10개로 확장할 수 있는 워크플로 시스템 구축

**현재 상황:**
- 본업: 제안서, 보고서, 마케팅 기획 등 문서 작업 (프리랜서/프로젝트 기반)
- 병목: 최종 디자인 작업에 시간 집중, 다른 개발 프로젝트와 충돌
- AI 활용 현황: 리서치에 많이 활용 중, 전체 작업의 40% 정도 AI 지원
- 도구: 옵시디언 사용 (태그 활용), HTMLtoPPT 스킬 인지하고 있음

**핵심 니즈:**
1. 맥락 전달의 어려움 해결 (클라이언트 내부 정치, 히스토리, 업계 특수 용어 등)
2. 레이아웃 고민 시간 단축 (콘텐츠 배치, 페이지 구성 미리 확인)
3. 프로젝트 수용량 10배 확장

---

## 설계한 워크플로 구조

```
[인풋 시스템] → [리서치] → [구조화] → [콘텐츠 생성] → [레이아웃] → [디자인] → [납품]
     ↓              ↓           ↓            ↓             ↓           ↓
  템플릿화        AI 주도     AI 주도      AI+검토       AI 초안      사람 집중
```

**예상 시간 변화:**
| 단계 | 현재 | 새 워크플로 |
|------|------|------------|
| 브리프 작성 | - | 15분 |
| 리서치 | 2시간 | 30분 |
| 구조화 | 3시간 | 30분 |
| 콘텐츠 작성 | 4시간 | 1시간 |
| 레이아웃 고민 | 3시간 | 30분 |
| 디자인 작업 | 8시간 | 4시간 |
| **총합** | **~20시간** | **~7시간** |

## 완료된 작업

### ✅ Phase 1: 브리프 템플릿 시스템 (완료)

옵시디언 마크다운 형식으로 5개 파일 생성:

1. **00_인테이크_체크리스트.md**
   - 통화/미팅 후 누락 정보 확인용
   - 자주 빠뜨리는 정보 TOP 5
   - 클라이언트 자료 요청 템플릿

2. **01_사업수주_제안서_브리프.md**
   - 경쟁 PT 상황, 평가 기준, 발표 형식 포함
   - 의사결정 구조 파악

3. **02_내부보고_기획서_브리프.md**
   - 보고 라인, 조직 문화 반영
   - 예상 Q&A 준비 섹션

4. **03_투자유치_IR_브리프.md**
   - 투자 라운드별 맞춤
   - 표준 IR 덱 구조 체크리스트
   - 약점 보완 전략 섹션

5. **04_마케팅캠페인_기획서_브리프.md**
   - 캠페인 유형별 대응
   - 채널 전략, 톤앤매너 정의
   - KPI 목표 설정

**모든 템플릿 공통 구조:**
- YAML frontmatter (태그, 상태 관리)
- 클라이언트 정보 + 의사결정 구조
- ⚠️ 맥락 정보 섹션 (놓치기 쉬운 부분 강조)
- 📅 일정 체크포인트
- 📦 제공받아야 할 자원 체크리스트
- 🤖 AI 협업용 요약 (복사해서 바로 전달)
- 작업 로그 + 회고

## 다음 진행할 작업 (우선순위 순)

### 🔲 Phase 2: 레이아웃 패턴 라이브러리
- 자주 쓰는 페이지 유형별 레이아웃 템플릿
- 예: 표지, 목차, 문제정의, 해결책, 데이터 비교, 팀소개, 타임라인 등
- React/HTML 아티팩트로 시각화하여 빠르게 선택 가능하게

### 🔲 Phase 3: AI 프롬프트 템플릿
- 각 단계별 표준 프롬프트
- 리서치 요청, 구조화 요청, 콘텐츠 생성, 레이아웃 제안 등

### 🔲 Phase 4: 실제 프로젝트 테스트
- 하나의 실제 프로젝트에 전체 워크플로 적용
- 병목 지점 파악 및 개선

## 사용자 특성

- 시각 디자인 + 기획 전문가 (실무 경험 풍부)
- 프로젝트 의뢰: 주로 구두(전화/미팅), 가끔 이메일/SNS
- 통화 후 정리하는 스타일 (실시간 메모 X)
- 클라이언트: 기업 (대기업~스타트업)
- 프로젝트 유형: 사업 수주 제안서, 내부 보고 기획서, 투자 IR, 마케팅 기획서
- 도구: 옵시디언 (태그 활용)
- 현재 월 1개 프로젝트 → 목표 월 10개
