# 내부용 실행 설계 (비전→액션 플랜·HOW TO)

> 뉴스/배경은 생략하고, **어떻게 실행할 것인가**에 초점.  
> 입력: 2차 목차(내부) + 리서치 요약 수치.  

## 1) 비전·목표 (요약만 유지)
- 분양률 24% → 60%(3년), 추가 80실 400억.  
- OCC 66→75, ADR 40→45, 영업이익 △171→+100, NPS 70.  
- 총투자 60.2억(브랜드/마케팅/운영) → ROI 1,220%.

## 2) 실행 원칙
- 고객·수익 직결 과제 우선: 분양/리드, OCC/ADR, NPS.  
- 스프린트 운용: 6주 단위(준비→런칭→검증).  
- 데이터 우선: CAC·리드전환·OCC/ADR·NPS를 주간 단위로 트래킹.

## 3) 스프린트 플랜 (6주 예시, 실행 항목 세부)
1. 준비(주1~2)  
   - 분양가/혜택 테이블 확정(타입별 가격·인센티브·금융 옵션).  
   - 투어/CTA SLA 정의(응답/투어/계약 소요, 담당자).  
   - 리드 소스 태깅 설계(채널 코드, 캠페인 ID) + 대시보드 골격.  
   - 패키지 SKU 초안(웰니스/패밀리/MICE) 가격·혜택·업셀 포인트.  
   - 디지털 최소 기능 범위 결정(PMS/RMS 연동, 앱/웹 예약, CRM 싱크).
2. 런칭(주3~4)  
   - 1:1 세일즈 플레이북 배포(스크립트, FAQ, 가격·혜택 테이블).  
   - 채널 온에어: VIP 1:1, 제휴(골프/계열), 프리미엄 매체, 디지털, PR/팸투어.  
   - 투어 운영 개시: 일정 슬롯·동선·체험 요소(뷰/스파/샘플 F&B) 세팅.  
   - 패키지 판매 시작: SKU별 견적서·계약서 템플릿 배포.  
   - NPS 샘플링 포맷 배포(체류 종료 24h 내 발송).
3. 검증(주5~6)  
   - 퍼널 점검: 리드→투어→계약 전환율, CAC, 채널별 기여도.  
   - OCC/ADR/부대수익 효과 검토, 단가·재고 조정.  
   - NPS/후기 수집 결과 리뷰, 서비스 개선 티켓 발행.  
   - 디지털 안정성 테스트(예약·결제·CRM 싱크), 오류 백로그 정리.
4. 리프레시(스프린트 말)  
   - 인센티브/가격 A/B 결과 반영, 채널 예산 재배분.  
   - SKU/업셀 번들 수정, 투어 경험 개선(체험 요소 추가/삭제).  
   - 다음 스프린트 목표·KPI 리셋.

## 4) 실행 모듈별 HOW TO
- 분양/세일즈  
  - 산출물: 타입별 가격·혜택 테이블, 1:1 스크립트, FAQ(금융/세제/예약), 계약·결제 플로우.  
  - 프로세스: 리드 접수 → 24h 이내 응답 → 투어 예약 → 투어(체험 포함) → 조건 협의 → 계약/금융.  
  - 관리: 리드 태깅, 전환율/리드 소스별 CAC, 주간 성과 리뷰.
- 채널/리드 운영  
  - 채널: VIP 1:1, 골프 제휴, 계열/인바운드, 프리미엄 매체, 디지털, PR/팸투어.  
  - KPI: 리드수·전환율·CAC·계약수, 채널별 ROI.  
  - 대시보드: 주간(리드/계약), 월간(OCC/ADR 영향), 캠페인별 성과 비교.
- 패키지/상품  
  - SKU: 웰니스(요가/숲 테라피/스파), 패밀리(8인 스위트+키즈), MICE/웨딩(회의+연회+숙박), 체험 패스(투어+스파/F&B 크레딧).  
  - 각 SKU 요소: 포함 서비스, 가격, 업셀(스파 추가, F&B 코스 업그레이드, 객실 업그레이드), 제한 조건(성수기/블랙아웃).  
  - 영업 자료: 견적서·제안서 템플릿, 옵션 리스트, 업셀 체크리스트.
- 운영/서비스  
  - SOP: 체크인/체크아웃, 투어 동선, 룸서비스 SLA, NPS 설문 발송·회수, VOC 티켓 처리 TAT.  
  - 인력: 식음·프런트 충원 계획, 교육(서비스 스크립트, 업셀 트레이닝).  
  - 품질: NPS 목표 70, 주간 VOC 리뷰, 재방문/추천률 추적.
- 디지털  
  - 롤아웃: PMS/RMS/예약/결제/CRM 연동 일정, 최소 기능 우선(예약·결제·리드 캡처).  
  - 모니터링: 예약 성공률, 오류율, SLA 알림(응답/투어/계약), 채널별 온라인 예약 비중.  
  - 데이터: 리드·예약·투숙 데이터 싱크, 캠페인 코드 자동 반영.

## 5) RACI (요약)
- 분양가/혜택: 재무(R) · 세일즈(A) · 마케팅(C) · 법무(I)  
- 리드/채널: 마케팅(R) · 세일즈(A) · CRM(C) · 재무(I)  
- 패키지: 운영(R) · 세일즈/마케팅(C) · 재무(I)  
- 디지털: IT(R) · 운영/세일즈(C) · 경영진(A)  
- KPI/대시보드: 데이터(R) · 세일즈/운영(C) · 경영진(I)

## 6) KPI 체계 (주간/월간/분기)
- 분양: 리드→투어→계약 전환율, CAC, 월 계약 수, 잔여 물량.  
- 운영: OCC, ADR, RevPAR, 부대수익 비중, 인건비율.  
- 경험: NPS, VOC 처리 TAT, 재방문/추천률.  
- 채널: 채널별 리드/계약/ROI, 제휴별 매출 기여.  
- 디지털: 온라인 예약 비중, SLA 준수율(응답/투어/계약), 오류율.

## 7) 리스크·대응 (행동 기준)
- 분양 부진: 인센티브·가격 A/B, 법인 패키지 프로모션, 채널 믹스 재배분.  
- 인지도 부족: PR·제휴 집중 스프린트, 레퍼런스 투어(계열/VIP) 운영.  
- 디지털 지연: 수동 백업(예약/리드), 최소 기능 우선 출시, 핫라인 운영.  
- 인력/서비스: 식음·프런트 충원, 서비스 재교육, VOC 주간 리뷰로 보상/휴식 조정.  
- 시설/CapEx: 단계별 집행, 우선순위(뷰/객실/스파) 명확화, 예비비 관리.

## 8) 즉시 실행 체크리스트 (이번 스프린트)
- [ ] 타입별 분양가·혜택·인센티브 확정, 세일즈 배포  
- [ ] 투어/CTA SLA(응답/투어/계약) 확정·공유  
- [ ] 채널별 리드 대시보드(주간) 오픈, 소스 태깅 적용  
- [ ] 패키지 SKU(웰니스/패밀리/MICE) 가격·혜택 확정, 업셀 포인트 정의  
- [ ] PMS/RMS/앱 최소 기능 출시 일정 확정, 예약/CRM 연동 테스트  
- [ ] NPS 수집·VOC 루프 가동(주간 리뷰 일정 포함)
